\section{Expected number of elements in a Bloom Filter}
In a Bloom Filter with $B$ bits set, the number of elements expected, $N$, is
\begin{equation*}
  N \approx - m \ln [1 - B/m]/k
\end{equation*}

Wikipedia claims proved in Swamidass \& Baldi (2007):

Swamidass, S. Joshua; Baldi, Pierre (2007), "Mathematical correction for fingerprint similarity measures to improve chemical retrieval", Journal of chemical information and modeling (ACS Publications) 47 (3): 952â€“964

There is almost certainly a better/earlier source on this.

The proof of this does assume each of the $N$ elements flipped on $k$ bits, which is not completely true and this bound can be made better taking into account collisions of bits of the elements put into the filter.
